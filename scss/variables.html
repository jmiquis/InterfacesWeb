Tipos de datos

Sass/SCSS permiten trabajar con muchos tipos de datos, muchos de los cuales provienen directamente de CSS:

  • Números, que pueden tener unidades o no, como 12 o 200px.
  • Strings, que pueden tener comillas o no, como "Helvetica Neue" o bold.
  • Colores, que pueden indicarse en hexadecimal (#F00), por el nombre (red), o como funciones (rgb(255,0,0) o hsl(0, 100%, 50%))
  • Listas de valores, que pueden estar separadas por espacios (1.5em 1em 0 2em), comas (Helvetica, Arial) o corchetes ([col1-start]).



y otros son más específicos de Sass/SCSS, siendo los más comunes:

  • Mapas que permiten asociar valores con claves, como ("background": red, "foreground": pink).
  • Booleanos, cuyos posibles valores serán true o false.
  • Null o ausencia de valor, muy útil para el retorno de funciones.

Variables

Sass usa el símbolo $, seguido de un nombre, para declarar variables:

Sintaxis variable de Sass:

$variablename: value;

Alcance variable de Sass

Las variables de Sass solo están disponibles en el nivel de anidamiento donde están definidas.

El comportamiento predeterminado para el ámbito de la variable se puede anular mediante el !globalconmutador.

!global indica que una variable es global, lo que significa que es accesible en todos los niveles.

h1 {
  $myColor: green !global;
  color: $myColor;
}

p {
  color: $myColor;
}

Sass te permite anidar selectores CSS de la misma forma que HTML.

Mientras que en CSS, las reglas se definen una por una (no anidadas).

Sass, es más limpio y más fácil de leer que el CSS estándar.

Muchas propiedades CSS tienen el mismo prefijo, como font-family, font-size y font-weight o text-align, text-transform y text-overflow.

En SASS:

font: {
family: Helvetica, sans-serif;
size: 18px;
weight: bold;
}

text: {
align: center;
transform: lowercase;
overflow: hidden;
}

Al transpilarlas en CSS quedarían así:

font-family: Helvetica, sans-serif;
font-size: 18px;
font-weight: bold;

text-align: center;
text-transform: lowercase;
text-overflow: hidden;
